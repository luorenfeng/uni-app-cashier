# 移动端扫码收银系统

基于 `Vue 3`、`uni-app` 和 `SQLite` 开发的 Android 端离线扫码收银应用，适用于小型门店、单机收银和商品快速录入场景。项目围绕“扫码识别、商品管理、订单结算、订单留存”构建完整业务闭环，强调移动端操作效率和弱网/离线环境下的可用性。

## 项目亮点

- 基于 `Vue 3 + uni-app` 实现 Android 端扫码收银流程，完成扫码识别、商品补录、商品库展示、商品编辑、订单结算和订单列表展示。
- 接入原生扫码能力，处理相机权限申请、失败提示、条码校验与异常分支，优化移动端扫码体验和交互稳定性。
- 使用本地 `SQLite` 持久化商品库与订单数据，支持离线商品查询、离线结算和订单保存。
- 实现商品库搜索、防抖过滤、数量加减、总价实时计算、回到顶部等交互能力，提升页面操作效率。
- 优化商品库迁移策略，区分预置商品与用户自定义商品，避免版本更新时误删用户数据。

## 功能说明

### 1. 扫码收银

- 调用 Android 原生扫码能力识别商品条码
- 已录入商品直接加入结算列表
- 未录入商品可在当前页直接补录名称与价格
- 支持同一商品数量累加、手动加减和总价实时计算

### 2. 商品库管理

- 查看本地商品库
- 按商品名称或条码搜索
- 编辑商品名称、价格
- 删除自定义商品

### 3. 订单管理

- 对当前扫码商品进行结算
- 自动生成本地订单记录
- 在订单页查看历史订单和商品明细

## 技术栈

- `Vue 3`
- `uni-app`
- `Composition API / script setup`
- `SQLite`
- `Vite`

## 本地存储方案

- 商品数据：本地 `SQLite`
- 订单数据：本地 `SQLite`
- 运行模式：以 Android 单机离线使用为主，不依赖服务端即可完成核心收银流程

## 快速启动

### 安装依赖

```bash
npm install
```

### 构建 Android 应用

```bash
npm run build:app-plus
```

构建完成后，可使用 HBuilderX 导入 `dist/build/app` 继续运行、调试或打包 APK。

## 运行要求

- 需要在 `manifest.json` 中启用 `Barcode`、`Camera`、`SQLite` 模块
- 首次扫码时需要授权相机权限
- 当前项目主要面向 Android 端使用

## 适用场景

- 小型门店收银
- 社区零售
- 临时展销
- 单机离线商品管理与订单记录
